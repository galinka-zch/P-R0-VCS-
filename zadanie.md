сведения о себе 
git config --global user.name "    "
git config --global user.email "    "

создать локальный репозиторий (где храниться на компе будет)
 cd путь к папке
(cd c:/gitzadanie)

git init -  инициализируемся, создаётся в текущем каталоге новый подкаталог(скрытая папка) с именем .git содержащий все необходимые файлы репозитория — основу Git-репозитория

git remote add <имя репозитория> <адрес из githab> - добавить новый удалённый Git-репозиторий под именем-сокращением
(git remote add P-R0/VCS https://github.com/galinka-zch/P-R0-VCS-.git)

добавить файл в репозиторий локальный
touch <file.md (php, txt)>
(touch zadanie.md - файл появиться в папке)

изменяем в папке на компе этот файл, далее добавляем этот файл

git add <file.md (php, txt)>
 добавляет содержимое рабочей директории в индекс для последующего коммита
(git add .     - добавляет все в текущей директории
git add *.txt - добавляет все тхт файлы в текущей директории)

git commit -m 'осмысленное указание изменений'
вводим логин
       пароль

git push - отправляем все в удаленный репозиторий (заливаем все)

git clone <url - адрес для клонирования берем из githab>

git status - для того, чтобы увидеть текущее состояние 
нашего проекта

git log - журналов коммитов
git push -u origin master -заливаем все по умолчанию на githab
git pull origin master -заливаем себе на комп (локальный репозитарий) все из гитхаб
git diff HEAD - посмотреть конкретные изменения HEAD – это указатель на текущую ветку, которая, в свою очередь, является указателем на последний коммит, сделанный в этой ветке. 
git diff  -разница между вашей рабочей директорией и индексом
git diff --staged -разница между индексом и последним коммитом
